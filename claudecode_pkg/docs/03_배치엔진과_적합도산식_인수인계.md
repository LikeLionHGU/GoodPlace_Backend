## 배치 엔진이 푸는 문제

공실 여러 곳에 업종을 배정하되 같은 업종을 두 공실에 넣지 않는다(배정 문제). ① V×I 점수 행렬을 만들고 ② 업종 중복 없이 총점 최대 조합을 찾는다.

## 3단계 — 적합도 산식 ✅ 확정 (engine.py)

```
score = demand × area_fit × competition_factor
  demand             = 직접 투표수 + w1 × 인근 공실(반경 500m, 하버사인 거리)의 같은 업종 투표수
  area_fit           = 면적 범위 안 1.0, 벗어나면 비율 감점(하한 0.2)
  competition_factor = 동네 평균 대비 비율 기반:
                       ratio = 경쟁점수 / 그 업종 동네평균
                       ratio ≤ 1 → 1.0(감점 없음) / ratio > 1 → 1/(1 + w2×(ratio−1))
```

**가중치 확정 + 근거(Q3-2 방어용):**
- `w1 = 0.4` — 직접 표는 1차 신호, 인근 표는 절반 이하의 보조 신호
- `w2 = 1.0` — 동네 평균의 2배에서 수요 반토막(계수 0.5). 정상 밀도는 존중, 명백한 포화만 감점
- 인근 = 반경 500m

**경쟁을 동네 평균 대비 비율로 보는 이유:** "카페 3곳(흔함) ≠ 세탁소 3곳(드묾)". 절대 수로 깎으면 업종별 정상 밀도를 무시하게 된다. 분해(breakdown)에 `competition_ratio`(평균 대비 배수)를 노출하고, 리포트 포화 플래그가 이 값을 쓴다.

**산식의 성격 = 후보 필터.** 매출·성공 예측 모델이 아니라 "이 자리를 후보로 볼지" 거르는 1차 신호. 진짜 창업 근거는 리포트 전체가 만든다(과대약속 금지). 응답에 항상 점수 분해 7요소(direct_votes, nearby_weighted, area_fit, competitor_count, neighborhood_avg, competition_ratio, competition_factor)를 동봉한다.

## 4단계 — 겹침 해소 배치 ✅ 확정 (engine.py)

**헝가리안 알고리즘(scipy `linear_sum_assignment`) 채택.** 최대화 문제라 점수를 음수로 뒤집어 푼다. 근거: 공실 수십~수백에도 최적을 빠르게(전수탐색은 8곳부터 폭발). "표준 최적화 알고리즘 사용"으로 심사 방어.

환경 자동 폴백: scipy 있으면 헝가리안 → 없으면 전수탐색(최적 보장, 순열 20만 이하) → 초과 시 그리디. 현 개발환경은 scipy 미설치라 `exhaustive(최적)`로 동작 중이며 결과는 헝가리안과 동일(둘 다 최적해). 응답에 차순위(runners_up)·사용 가중치·실제 알고리즘을 항상 포함.

## 표시 적합도 % — 포화 곡선 통일 ✅ 확정 (report.py)

이전의 '4요소 가중합(수요40/경쟁30/입지20/면적10)' 표시는 **폐기**됐다. 곱셈(엔진)과 덧셈(표시)은 요소가 같아도 순위가 뒤집힐 수 있어(반례: 수요10×적합1.0=10 vs 수요100×적합0.2=20) 지도와 리포트가 다른 업종을 가리키는 자기모순이 생기기 때문.

```
표시 % = 100 × s / (s + K)     (s = 엔진 곱셈 점수, 단조 변환 — 요소 재조합 금지)
```
- **100%는 구조적으로 안 나온다**(내림 처리로 ≤99 보장) — "AI는 성공을 보장하지 않습니다"와 정합
- **K는 자의 상수 금지.** 기준 케이스(캠페인 목표 투표수 달성 + 면적적합 1.0 + 경쟁 평균 수준)를 `engine.score`에 실제로 통과시켜 검산한 값. 현재 `BASELINE_TARGET_VOTES = 30`(잠정) → K = 30
- **입지 20%는 %에서 제거** — 엔진 산식에 입지 항이 없으므로 통일의 자연스러운 결과. 층수는 참고 정보로만 표시
- 리포트 1순위 = `allocate` 배정 업종(재랭킹 금지, 지도 핀과 일치). 배치 근거 값은 engine breakdown을 그대로 씀

## 남은 미확정 (팀 결정 — 확정처럼 쓰지 말 것)

- **K 전역 고정 vs 캠페인별:** 캠페인별이면 지역 간 % 비교가 무의미해짐(같은 60%가 다른 수요). 현재 전역 고정 '잠정'
- **BASELINE_TARGET_VOTES = 30 잠정:** 양덕동 실데이터 확보 후 캘리브레이션(절차는 report.py 주석)
- **층수 계수의 '자리':** 1층 vs 상층은 논문상 공실과 강한 관련이 있는 유일한 입지 실데이터 변수. 산식에 켤지(예: 상층 감점) / 참고 정보 유지할지 미확정
- 확장 요소(인구 대비 적정성 등)는 데이터 소스 확보 시 검토(04번 API 검증 결과: 인구·유동인구는 소상공인 API에 없음)
