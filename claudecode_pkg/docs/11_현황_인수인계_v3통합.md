## 2. 완료된 것

**A 데이터계층 (I0 · 채택·검증·push):**
- votes v3(region_code+industry_id+voter_grid, 공실 아님), 다중선택 `/votes/batch`+냥 일괄차감,
  `snap_to_grid`(200m), 동네수요 `/regions/{code}/demand`, 냥 캐시(credit/use/balance, 10원=1냥),
  placements 테이블, 소상공인 ingest(`/admin/ingest`).
- 우리 스택 통일: ingest `httpx`·env `SBIZ_SERVICE_KEY`, main `.env` 로딩.

**B 엔진 (I1 · 완료·검증·push):**
- `score = demand × area_fit × competition_factor × floor_fit`
- demand = 격자별 표수 × 거리감쇠(≤500=1.0 / 500~800 선형 / >800=0)
- floor_fit: 1층 1.0 / 2층이상 0.7 / 지하 0.5 / 미상 1.0 (config.FLOOR_FIT)
- 경쟁 = vacancy.competitors[업종명] 대비 동네평균(w2=1.0). 헝가리안 배치·차순위 유지.
- 검증: test_engine_v3(19) — 거리감쇠 경계·단조, floor_fit, 손계산, 배치 겹침.

**리포트·AI해설·캠페인·지도 (I2~I5 · 완료·검증·push, 상세는 §3 표):**
- `report.build_report`: 업종→공실 1·2·3위, 포화 곡선 %(`_saturation_k()`로 프로그램적 산출), floor 근거.
- `ai_explain`: `facts_from_report(report, card)` v3 형태, `POST /report`(50냥 차감·잔액부족 비차단).
- `campaign`/`routes_campaign`: region 기반 환불(기한 경과+미성사 → held 전건 환불, 멱등).
- `routes_map`: placements 파생 상태(vacant/preparing/open).
- `tests/run_all_v3.py`(일괄 러너)·`demo_v3.py`(원플로우 콘솔 데모) 정리 완료 — 전 회귀 135/135 green.

**아직 v1 스테일: `adapters.py`** — 현재 어디서도 import되지 않는 미사용 파일. 후순위 항목으로 남겨둠.

## 4. 컨벤션 (통합 확정)

- **id INTEGER**, **DB meongdang.db**, 키는 **`.env`**(OPENAI_API_KEY·SBIZ_SERVICE_KEY) — **넣음 완료**.
- **격자**: `snap_to_grid` → `"lat,lng"` 3소수 문자열(원좌표 미저장). 엔진이 파싱해 거리 계산.
- **냥**: 10원 = 1냥(A `won_to_nyang`). 투표 1건 = 1,000원 = 100냥. 리포트 생성 = 50냥(예정).
- **경쟁 competitors**: `{업종명: 수}`(A ingest 형태). 엔진은 업종명으로 조회.
- **floor**: 카테고리 `'1층'/'2층이상'/'지하'`(중개사 등록).
- 투자·펀딩 금지 → 선결제·쿠폰. 시드 `is_seed=1`. 결제 모의.
