# 04. API·데이터 인수인계

상세 명세는 `소상공인_상가정보_API_명세서.md` 참조. 이 문서는 인수인계용 요약.

## 사용 API: 소상공인 상가(상권)정보 (공공데이터포털)

- 서비스 ID: `B553077`
- 기본 URL: `http://apis.data.go.kr/B553077/api/open/sdsc2`
- 호출: `{기본URL}/{오퍼레이션}?serviceKey={인증키}&{파라미터}&type=json`
- 공통: `serviceKey`(필수), `numOfRows`, `pageNo`, `type`(json/xml)

## 우리가 이 API로 얻는 것

1. **양덕동 영업 점포 전수** → 업종별 개수 = 산식의 경쟁 밀도·주변 업종 구성
2. **폐업 점포** → 공실 후보 + 이전 업종 (`clbizDt` 폐업일자, `indsSclsNm` 업종)
3. **업종 247 코드 체계** → 투표 업종·점포 업종 매칭 기준

**핵심 한계:** '이 점포는 공실'이라고 직접 주는 필드는 없음. 공실은 폐업+현재영업+지도확인 조합으로 추정.

## ★ API 능력 검증 — 우리가 원하는 것 vs API가 주는 것

우리 서비스 요구사항별로 소상공인 API가 감당 가능한지 대조한 결과다.

| 우리가 원하는 것 | 판정 | 근거·방법 |
|---|---|---|
| 양덕동/경북 점포 목록 수집 | ✅ 가능 | `storeListInAdmi`(행정동)·`storeListInRadius`(반경)·`storeListInRectangle`(사각). 행정동/시군구 코드만 바꾸면 경북 전역. 단 대량은 페이징+캐싱 필요 |
| 업종 구성·경쟁 밀도 | ✅ 가능 | 점포별 업종코드+좌표로 반경 내 동일업종 수 계산 |
| 이전 매장이 무엇이었나 | 🔶 부분 | 폐업 레코드의 직전 업종(`indsSclsNm`)·폐업일(`clbizDt`) 확인 가능. 단 한 자리의 전체 순차 이력·소급 범위는 [확인필요] |
| 공실 여부 자체 | ❌ 직접 불가 | API는 '영업 중'이 기본. 폐업기록+영업목록 대조+로드뷰로 **추정** |
| 주변 인구수·인구 대비 적정성 | ❌ 불가 | 소상공인 API에 인구 없음 → **통계청 SGIS / 행안부 주민등록 인구** 별도 필요 |
| 유동인구 | ❌ 불가 | API에 없음 → 통신사 유동인구·SGIS 별도, 또는 MVP는 경쟁점 보정만으로 산식 성립 |

**검증 결론:** 점포·업종·경쟁·이전업종(부분)은 이 API로 해결. 공실 여부는 조합으로 우회. **인구·유동인구 두 축은 별도 공공데이터가 필요.** → 적합도 산식은 "데이터 확보 가능한 요소(투표·면적·경쟁·업종구성)부터 켜고, 인구·유동인구는 소스 확보 시 확장"하는 구조가 맞다(03번 문서와 일치).

## 경북 전체 확장 설계 (양덕동은 첫 적용지)

지역을 하드코딩하지 말고 **행정동/시군구 코드를 파라미터로** 받게 설계한다. `storeListInAdmi`의 `key`(행정동코드)만 바꾸면 양덕동 → 포항 타 지역 → 경북 전 시군구로 코드 변경 없이 확장된다. DB의 공실·점포에 지역 코드 컬럼을 두고, 배치·리포트가 지역 단위로 동작하게 하면 "양덕동에서 검증, 경북으로 확장"이 코드가 아니라 데이터로 이뤄진다. 주의: 경북 전체 점포는 대량이라 지역별 사전 수집·캐싱 전제(실시간 전국 로딩 아님).

## 추가로 필요한 데이터 소스 (소상공인 API 밖)

- **인구:** 통계청 SGIS(통계지리정보) 또는 행정안전부 주민등록 인구통계 — 행정동 단위 인구·연령. 산식의 '인구 대비 업종 적정성'용
- **유동인구:** SGIS 생활인구 또는 통신사 데이터 — MVP 필수 아님, 로드맵
- **건축물대장:** 공실 면적·층 보강 (부록 E 파이프라인)
- **로드뷰:** 카카오/네이버 — 공실 육안 확인

## 핵심 오퍼레이션 (양덕동·경북 1순위)

- `storeListInAdmi` — 행정동 코드 기준 점포 목록 (`divId=adongCd`, `key=양덕동코드`) ★
- `storeListInRadius` — 중심좌표+반경 점포 (`radius`, `cx`/`cy`)
- `largeUpjongList`/`middleUpjongList`/`smallUpjongList` — 업종 코드 조회

## 점포 응답 핵심 필드 (확인됨)

- 식별: `bizesId`(점포번호), `bizesNm`(상호)
- 업종: `indsLclsNm`/`indsMclsNm`/`indsSclsNm`, `ksicCd`(표준산업분류)
- 위치: `rdnmAdr`(도로명), `lnoAdr`(지번), `bldNm`, `flrNo`(층), **`lon`/`lat`**
- 이력: **`opbizDt`(개업일), `clbizDt`(폐업일)**, `chgGb`/`chgDt`(변경)

## 양덕동 공실 특정 파이프라인

1. `storeListInAdmi`로 양덕동 점포 전수 수집
2. `clbizDt` 비어있음 = 영업 중 / 채워짐 = 폐업. 분리.
3. 영업분 → 업종 구성·경쟁 밀도 계산
4. 폐업분 → 공실 후보·이전 업종. 단, 폐업 주소에 현재 영업 점포 있으면 공실 아님(대조 필수)
5. 카카오/네이버 로드뷰 육안 확인 + 건축물대장으로 면적·층 보강 → 공실 확정

## 업로드된 파일 3종 (프로젝트에 있음)

| 파일 | 내용 | 쓰임 |
|---|---|---|
| OpenApi 활용가이드(hwp) | API 오퍼레이션·파라미터·필드 정의 | 명세서의 원본 |
| 업종분류(2302) 연계표(xlsx) | 247개 업종코드 3시트(분류/구신연계/KSIC연계) | 투표 업종↔점포 매칭 기준표 |
| 주요상권현황 CSV(20240101) | 전국 1,227개 상권 폴리곤. **CP949 인코딩** | 상권 영역 참고 |

**CSV 파싱 주의:** `encoding='cp949'`, 상권좌표 필드가 길어 `csv.field_size_limit(sys.maxsize)` 필요.

**중요 발견:** CSV에 **'양덕' 상권 없음** (포항 25개 중). 양덕동은 '주요상권' 밖 → 상권 단위가 아니라 **행정동/반경 조회로 접근**해야 함.
