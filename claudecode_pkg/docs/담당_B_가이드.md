# 담당 B 가이드 — 엔진·산출 파트

**먼저 `AB공통_분담과통합계약.md`를 읽고 이 문서를 본다.** 경계·스키마·인터페이스 계약은 공통 문서가 기준이다.

## B의 책임 한 줄

데이터가 **나가는** 쪽 전부. A가 채운 데이터를 받아 배치 엔진으로 계산하고, 창업 리포트를 만들고, 캠페인·환불 상태를 판단한다. **B의 엔진은 순수 함수**라 DB 없이도 더미 데이터로 개발·테스트할 수 있다. 이게 A와 독립적으로 일할 수 있는 이유다.

## B가 만드는 단계

### 3단계 — 적합도 산식 ✅ 확정·구현됨 (engine.py)
- 순수 함수. `score = demand × area_fit × competition_factor`
  - demand = 직접 투표수 + w1 × 인근(반경 500m, 하버사인) 같은 업종 투표수
  - area_fit = 면적 범위 안 1.0, 벗어나면 비율 감점(하한 0.2)
  - **competition_factor = 동네 평균 대비 비율식**: ratio ≤1 → 1.0 / ratio >1 → 1/(1+w2×(ratio−1))
- **가중치 확정: w1=0.4**(인근은 보조 신호), **w2=1.0**(평균 2배에서 반토막). 되돌리려면 사람 확인
- 확장 요소(인구 등)는 데이터 확보 시(04번: 소상공인 API에 인구·유동인구 없음)
- 검증 통과: 감점 단조성, 손계산 일치, 투표 0→점수 0, breakdown 7요소 노출

### 4단계 — 겹침 해소 배치 ✅ 확정·구현됨 (engine.py)
- `allocate(...)` 순수 함수. **헝가리안(scipy) 확정** — 폴백: 전수탐색(최적)→그리디 자동
- 응답에 점수 분해·차순위·사용 가중치·실제 알고리즘 포함 ("왜 이 배치냐" 방어)
- 검증 통과: 극단케이스(전 공실 카페 1위) 겹침 없음, 전수탐색 ≥ 그리디

### 5단계 — 창업 리포트 (5요소) 🔶 코드 완성(report.py·포화 곡선 구현) — API 노출만 남음
- **`07_창업리포트_스펙.md`(v2)가 확정 기준.** 시안은 `명당_창업리포트_예시.html`.
- `build_report(...)` 순수 함수. 필수 5요소(순서 고정): ①한 줄 결론+적합도 ②대기 고객 수(맨 앞·강조) ③배치 근거(곱셈 3요소·포화 곡선 %) ④주변 경쟁(비교 기준선 필수) ⑤참고 비용+인허가
- LLM 미사용, 서버가 결정적으로 채움(챗봇 없음 = 환각 통제). LLM 붙여도 구조 유지
- **정보 과잉 금지:** 시간대별·성별연령·비용 7단계 분해 등은 넣지 않는다(스펙 문서 '빼는 것' 참조)
- **창업 비용은 참고값만:** 업종 평균+출처+단서, 정밀 예측 금지
- **출처 태그(실측/API/예시/업종평균)** 로 값마다 출처 구분 — 소상공인 API로 못 얻는 값(보증금·유동인구)은 '예시'
- 검증: 5요소 채움, 수치가 투표·배치 결과와 교차 일치, 없는 공실 404, 태그·비교 기준선 존재

### 6단계 — 캠페인·환불 로직 (모의)
- 캠페인 기한 판단, 기한 경과 시 미성사 투표를 **캐시 환불(모의)**로 상태 전이
- 실제 캐시 원장 기록은 A의 `cash_ledger`에 위임 — B는 "어떤 투표를 환불 대상으로 판단"까지, 기록은 A 함수 호출
- 검증: 기한 전 환불 0, 기한 후 대상 전건 전이, 환불분이 집계·배치에서 제외

## B가 반드시 지킬 것

- **엔진은 순수 함수.** DB 직접 접근 금지. 인자로 받은 vacancies·industries·vote_counts만 계산 → A와 독립, 테스트 쉬움
- 공통 문서의 함수 시그니처 고정: `allocate(...)`, `build_report(...)`
- vote_counts는 `(vacancy_id, industry_id)→투표수` 형태로만 받는다 (votes 원본 구조 모름)
- 배치 결과에 근거(점수 분해) 항상 포함 — 심사 방어 자료

## B의 결정 대기 (팀·A와 조율)

- 산식 최종 요소·가중치 w1·w2 (03번)
- 배치 알고리즘: 헝가리안 vs 공실 수 제한 (03번, A의 공실 수집 규모 보고)
- 스키마 최종 확정 (A와 합의) — 특히 vacancies의 `prev_industry`·`competitors` 형태
