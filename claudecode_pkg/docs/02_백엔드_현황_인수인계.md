# 02. 백엔드 현황 인수인계 (v2 — mini-main 코드 기준)

**2026.07.14 개정.** "코드 0부터 재시작" 선언 이후 Claude Code가 `mini-main` 저장소에 실제 구현을 진행했다. 이 문서는 그 코드베이스의 검증된 현황이다. **문서와 코드가 다르면 코드가 우선.** 작업 규칙의 원본은 저장소의 `CLAUDE.md`.

## 기술 스택 (확정)

Python + FastAPI + SQLite(mydang.db) + uvicorn. scipy는 선택 의존(있으면 헝가리안, 없어도 동작). 테스트는 stdlib만으로 실서버 기동까지 검증.

## 아키텍처

```
[클라이언트] → [FastAPI 라우터] → [어댑터] → [엔진(순수함수)]
                routes_*.py       adapters.py   engine.py / report.py
                                     │
                               [SQLite mydang.db]  db.py
```
엔진은 DB를 모르고(순수 함수), A·B의 유일한 접점은 집계 뷰 `vote_counts_view`가 보장하는 `(vacancy_id, industry_id)→투표수` 계약 형태다(held/settled만 집계, 환불 표 제외).

## 시드 (seed_dummy.py — 전부 is_seed=1/SEED)

업종 6종: cafe(카페)·bakery(베이커리)·bunsik(분식)·book(서점문구)·banchan(반찬가게)·fruit(과일가게), 각각 면적 범위·평균 창업비용·**인허가(licenses)** 포함. 공실 4곳: V-A~V-D(양덕동 예시 좌표, region_code=4711158000, 직전 업종·경쟁점 JSON 포함). 시드 투표는 여러 공실에서 카페가 1위가 되게 설계 — 겹침 해소가 실제로 동작함을 보이기 위함. ※ 값은 발표 승인 전 임시.

## ⚠️ 알려진 이슈 3건 (다음 작업 대상)

1. **voter_name 마이그레이션:** 동봉된 mydang.db에 `votes.voter_name` 컬럼이 없음(CREATE IF NOT EXISTS 한계). 이 DB로 POST /votes 시 500. → ALTER TABLE 경량 마이그레이션 또는 DB 삭제 후 재시드.
2. **demo.py 스테일:** 리포트 ③ 출력에서 폐기된 4요소 가중합 키(`weight`, `note`)를 참조해 KeyError로 죽음. 시연 스크립트이므로 최신 report.py 출력(곱셈 3요소, `disclaimer`)에 맞춰 수정 필요.
3. **README.md 스테일:** 이미 확정된 것들(산식 통일·w2=1.0·헝가리안)을 여전히 "결정 대기"로 표기 — "코드가 진실" 원칙과 어긋남.

## 스키마: AB 계약 대비 추가 3건 (A·문서 통지 대상)

`industries.licenses`(리포트 인허가), `campaign.is_seed`(시드 표기 요건), `votes.voter_name`(고객 명단용). id는 TEXT형(예: "cafe", "V-A"). → AB공통 계약 문서 v2에 반영됨.
