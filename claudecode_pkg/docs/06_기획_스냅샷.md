# 06. 기획 스냅샷 v2 (동네 투표 모델 — 2026.07.14 전면 개정)

**용도:** 명당의 기획 전체를 한 파일로 복원하는 압축본. v1(공실 투표 모델)은 폐기, 이 v2가 유효.
상세 명세는 08(리포트·공실등록·산식), 원본 검증 자료는 프로젝트 지식의 기획서·5Whys·인터뷰.

---

## 문제 정의 (유지)

경북 동네 상가 공실이 장기화되는데, 예비창업자가 '이 동네에 무엇이 필요하고 실제로 돈이 될지' 판단할 근거가 없어 입점을 주저 → 공실 지속. **근본 원인은 주민의 미충족 수요를 신뢰 가능한 형태로 수집해 창업자에게 전달하는 채널의 부재.** 병목은 임대료가 아니라 수요 불확실성 + 집객("임대료 내려도 안 나간다" — 부동산 인터뷰).

## 솔루션 한 줄 (v2)

주민이 1,000원으로 '생기면 쓸 3,000원 쿠폰'을 선구매하며 **우리 동네에 원하는 업종**에 투표하면, AI가 그 수요를 읽어 **업종별로 최적의 빈 상가 위치를 추천**하고, 창업자에게 리포트·첫 손님 명단을 연결한다.

## 서비스 플로우 (v2 — 투표 대상이 동네로 바뀜)

```
[주민]   ① 동네 수요 리스트 확인 → ② 원하는 업종에 1,000원 투표(=3,000원 쿠폰 선구매)
         투표 시점 GPS를 '행동 범위'로 저장(200m 격자, 주거지 주장 아님)
[창업자] ③ 주민과 같은 동네 수요 리스트(업종별 득표 순위)를 봄
         ④ 업종 클릭 → AI가 그 업종의 추천 공실 1·2·3위 + 리포트 제시
[AI]     수요(거리 감쇠 가중) × 면적 적합 × 경쟁 보정으로 공실별 점수
         → 지도엔 동네 전체 최적 배치(헝가리안), 리포트엔 업종별 공실 순위
         → 확정 수치를 LLM이 창업자 언어로 해설(환각 억제 구조)
[성사]   창업 결정 → 중개사 연결(계약은 중개사) → 개업일 쿠폰 사용 = 첫 손님
[불발]   캠페인 기한 경과 시 1,000원 캐시 적립(모의) → 재투표·쿠폰 구매에 사용
```

## 왜 동네 투표인가 (전환의 근거)

- 주민 행동으로 자연스럽다: "빈 상가 목록에서 골라 투표"(부자연) → "우리 동네에 뭐가 있으면 좋겠다"(자연).
- AI의 역할이 선명해진다: 수요 수집(사람)과 공실 매칭(AI)이 분리 → 위치 추천이 AI의 일이 됨.
- **유동인구 약점이 강점으로**: 유동인구 추정치 대신 "이 동네를 실제로 지나다니며 지갑을 연 주민 수(행동 범위 수요)"를 센다.

## AI 적용 지점 (정직한 3단 프레이밍)

1. **구조화(로드맵/stretch):** 자유 텍스트 수요를 업종으로 분류·군집(LLM) — MVP 밖.
2. **최적화(구현):** 거리 감쇠 수요 계산 + 헝가리안 배치(지도용 전체 배치) + 업종별 공실 순위. 규칙·최적화 알고리즘임을 정직하게 명명.
3. **생성(구현 확정):** 리포트 자연어 해설 — **GPT API(OpenAI, 팀 보유 키)**. 입력은 우리가 계산한 확정 수치 JSON만, 출력은 3~5문장. 새 숫자·사실 생성 금지, 성공 보장 표현 금지(환각 억제 프롬프트). "AI를 쓰되 통제한다"가 시연 포인트.
4. **예측(로드맵):** 캠페인 누적 후 투표→실방문 전환율 학습 — 우리만의 데이터 자산. 지금은 하지 않음(학습 데이터 없음).

## 수요 산식 (v2 — 08번 §2가 원본)

```
공실 수요 demand = Σ 투표자별 거리 가중   (해당 동네·해당 업종 투표자)
  d ≤ 500m → 1.0 / 500~800m → 선형 감소 / >800m → 0   (설계 초기값, 실데이터로 조정)
score = demand × area_fit × competition_factor(동네 평균 대비, w2=1.0)
표시 % = 포화 곡선 100×s/(s+K) — 단조 변환, 100% 미도달
```
구모델의 w1(직접/인근 가중)은 폐기(공실 직접 투표 소멸).

## 창업 리포트 (v2 — "이 업종에 어느 공실")

업종 클릭 시: 한 줄 결론 + 동네 전체 수요(N명) → 추천 공실 1·2·3위 탭(공실별: 적합도%·행동범위 수요·경쟁·면적·업종가능 3층 판정·대기 고객+쿠폰 가치) → AI 해설(확정 수치 기반 3~5문장) → 참고 비용·인허가 → "답하지 않는 것" 체크리스트 → DISCLAIMER + 중개사 CTA. 유동인구는 행동 범위 수요로 대체. 정보 과잉 배제, 보류 3건 유지.

## 공실 등록 (v2 신규 — 중개사 협력의 실체)

공실 정보 = API/실측(위치·면적·직전업종·경쟁) + **중개사 등록**(층·건물용도·간이 설비 체크박스·임대조건, 출처 태그 '중개사등록'). 업종 가능 판정 3층: 면적(지금) / 중개사등록(등록 시) / 상세 설비(확인 항목). 직방·다방의 매물 옵션과 같은 입력 구조 — 우리는 그것을 창업 적합성 판정에 쓴다.

## 1,000원 설계·법적 포지셔닝 (유지)

투자·펀딩 금지 → 선결제·쿠폰 선구매. 불발 시 캐시 적립(모의). 결제 전체 모의(실PG·사업자 없음). 법률 검토 이력 없음 — "설계 방향"으로만.

## BM (유지) 

성사 수수료 / 점포 단위 심화 리포트 / B2G 수요 리포트(POPEN 회신 대기). 주민 무료. + 로드맵: 전환율 데이터 자산.

## 검증 현황 (유지 — 수치·출처 관리 동일)

설문 26명 긍정 92.3%(제출본 24명·83% 병기), 부동산 4곳·매장 2곳 인터뷰, 반대 데이터 공개 원칙. 사용 금지 수치("구미 24.4%"·"이면도로 60%") 유지.

## 미확정 (확정처럼 쓰지 말 것)

거리 감쇠 파라미터(500/800/선형) 실데이터 검증 · K/목표 투표수 · 캐시 3건 · 층수 계수 · 중개사 등록 UI를 MVP에 넣을지(시드 대체 가능) · 용도지역 API(브이월드·건축물대장) 보강 여부.
