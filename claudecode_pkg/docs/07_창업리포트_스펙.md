# 07. 창업 리포트 확정 스펙 (담당 B 5단계 구현 기준)

> ⚠️ **부분 구판(2026.07.14 동네 투표 전환).** 리포트 구조('이 공실에 어느 업종')는 **08번 §3으로 대체**('이 업종에 어느 공실' + AI 해설). 근거 게이트·보류 3건·예측 3단계·비용 취급·출처 태그·정보 과잉 배제 원칙은 유효.


이 대화에서 확정한 명당 창업 리포트의 내용과 형식. 담당 B의 5단계(리포트 생성) 구현은 이 문서를 기준으로 한다. 배경 분석은 `나이스비즈맵_레포트_구조분석.md`, 시안은 `명당_창업리포트_예시.html`.

## 설계 원칙: 적게, 결정적으로

창업자가 리포트를 보는 이유는 단 하나 — **"이 자리에서 이걸 시작해도 될까?"** 이 결정에 직접 기여하지 않는 정보는 소음이다. 나이스비즈맵은 26장·수십 지표지만 대부분 '영업 중인 가게'의 운영 최적화용이라 창업 전 단계엔 안 맞는다. **정보량으로 경쟁하지 않는다. 우리만 줄 수 있는 것(미래 수요·대기 고객)으로 차별화하고, 나머지는 결정에 필요한 최소로 압축한다.** 목표는 스크롤 1~2번의 실질 한 장.

## 필수 5요소 (이 순서로)

1. **한 줄 결론 + 종합 적합도.** 맨 위에 "이 공실에는 세탁소, 적합도 82%"를 먼저. 계속 볼지 여기서 판단하게.
2. **검증된 수요 = 대기 고객 수.** "선결제 32명, 개업일 첫 손님." 우리만 주는 데이터이자 리포트의 심장. 시각적으로 가장 튀게(시안에서 골드 카드).
3. **배치 근거 (AI 판단).** 엔진 곱셈 3요소(수요 / 면적 적합 / 경쟁 보정)의 분해를 engine breakdown 그대로. 층수·직전 업종은 참고 정보로만. 이 수준이 신뢰를 주는 최소이며, 더 파고들면 과잉.

**표시 적합도 % 규칙 (✅ 확정 — report.py 구현됨, 03번 v2 상세):** 이전의 '4요소 가중합(수요40/경쟁30/입지20/면적10)'은 곱셈 엔진과 순위가 뒤집힐 수 있어 **폐기**. 표시 % = 엔진 점수의 포화 변환 `100×s/(s+K)` (단조, 요소 재조합 금지). 100%는 구조적 미도달(내림 ≤99) — "성공 보장 안 함"과 정합. K는 기준 케이스(캠페인 목표 투표수, 잠정 30)를 engine.score로 검산한 값. 입지 20%는 %에서 제거(엔진에 입지 항 없음).

**포화 플래그 (✅ 확정):** competition_ratio로 기회(≤1)/주의(1<r<2)/포화(≥2). veto 아님 — 보여주되 경고(주민 유료 수요를 시스템이 무시하지 않음). 주의/포화면 경고를 ①한 줄 결론 옆에 동반(스크롤 전에 보이게).
4. **주변 경쟁.** 반경 내 동일·유사 업종 수. 소상공인 API 실집계. **비교 기준선(동네 평균) 필수.**
5. **참고 비용 + 인허가.** 업종 평균 창업비용(출처+단서) + 필요 인허가. 접어두는 보조 정보.

책임 범위 선긋기 문구 고정: "AI는 성공을 보장하지 않습니다. 검증된 수요와 개업일 첫 손님까지가 명당의 책임이며, 이후 운영·품질은 창업자의 몫." (기획서 가설3 보완 + 과대약속 방지)

## 빼는 것 (넣고 싶은 유혹을 참을 것)

나이스비즈맵에 있으나 우리는 **뺀다**: 시간대별·요일별 매출 집중도, 성별·연령별 3시점 비교, 결제 단가 추이, 비용 구조 7단계 분해, 인기 메뉴 TOP3. 이유 두 가지 — ① 영업 중 가게의 운영 정보라 창업 전 단계에 부적합 ② 우리가 정밀하게 줄 수 없어 억지로 넣으면 출처 약한 숫자만 늘어남.

## 창업 비용 취급 (중요)

**정밀 예측 불가, 참고값만.** 우리 데이터로 가능/불가:
- 가능(참고 수준): 업종 평균 창업비용(소상공인시장진흥공단 통계), 지역 평균 임차료, 매출 대비 비용 비율
- 불가: 특정 공실의 실제 보증금·월세(협상 사안), 권리금(공개 데이터 없음), 정밀 인테리어비
- **표기 원칙:** "약 4,500만원(업종 평균)" + "실제는 점포·권리금·인테리어에 따라 크게 다름, 참고값" 단서 + 출처. 정밀 예측처럼 포장 금지.
- 화법(질문 시): "공공 통계 기반 업종 평균을 참고값으로 제시. 권리금·실제 임차료는 공개 데이터가 없어 정밀 예측이라 주장하지 않음. 리포트의 핵심 가치는 비용 예측이 아니라 그 비용을 들일 가치가 있는지 판단하게 하는 검증된 수요."

## 데이터 출처 태그 (시연 정직성 = 가상 데이터 지양 요건)

리포트의 모든 값에 출처 태그를 붙여 화면에서 구분한다.
- **실측:** 팀이 현장/로드뷰로 확인 (면적·층 등)
- **API:** 소상공인 상가정보 API 확인값 (직전 업종·공실 기간·경쟁점 수)
- **예시:** 시연용 예시값 (보증금·유동인구 등 공개 데이터 없는 항목)
- **업종평균:** 공단 통계 평균 (초기비용)

하단에 범례를 두고, 시연에서 "무엇이 실데이터인지"를 화면 자체가 증명하게 한다. 소상공인 API로 못 얻는 값(보증금·유동인구)은 반드시 '예시' 태그.

## 비교 기준선 (나이스비즈맵에서 배운 것)

단독 숫자를 쓰지 않는다. 경쟁점 수 옆에 "동네 평균 X곳", 필요 시 전월 대비 등. "세탁소 0곳"이 아니라 "0곳 (동네 평균 2.1)"로 제시해야 기회인지 판단됨.

## 기획서 5섹션과의 대응

기획서 §6-3의 5섹션(①추천 업종·근거 ②예상 초기비용 ③필요 인허가 ④주변 경쟁 ⑤대기 고객)을 유지하되 **순서를 바꿈**: 대기 고객(⑤)을 결론과 함께 맨 앞으로. "132명이 기다린다"가 우리 정체성이므로. ③④는 소상공인 API로, ①⑤는 우리 고유 데이터(투표·배치)로 채움. = "그들의 상권 분석 + 우리의 수요 검증"을 한 장에.

## ★ 근거 게이트 (v4 — 전 기능 공통 원칙)

**모든 결과물은 데이터가 명확한 근거가 되어야 하며, 제대로 된 근거를 제공하지 못하는 기능은 보류한다.** 리포트에 새 항목을 넣기 전 반드시 통과할 질문: ① 이 값의 데이터 소스가 실재하는가 ② 출처를 화면에 태그로 달 수 있는가 ③ "근거가 뭐냐"는 질문에 한 문장으로 답이 되는가. 하나라도 아니면 보류 목록으로.

**감사 결과 (2026.07.14 전수 점검):**
- **유지(근거 명확):** 대기 고객 수(실투표 집계) · 쿠폰 확정 가치(투표수×3,000 산수) · 배치 근거 3요소(엔진 breakdown) · 헝가리안 배치(수학적 최적) · 주변 경쟁+동네 평균(소상공인 API, 현 SEED 태그) · 인허가(법령) · 유동인구 정직한 부재 · 체크리스트(데이터 주장 없음) · 시나리오 계산기(산수, 가정 주체=창업자)
- **조건부 유지:** 적합도 %(예측 아닌 엔진 점수의 지수 변환 — '지수' 라벨 유지, K 미확정 표기 유지) · 가중치 w1·w2·반경 500m('설계 초기값' 명시, 실데이터 확보 후 민감도 검증 조건)
- **🚫 보류(근거 게이트 미통과):**
  1. **지역 특성 노트** — 상권 유형은 용도지역 데이터, 주 연령대는 인구 데이터 필요(둘 다 미확보 · 04번: 소상공인 API에 인구 없음). 인터뷰 특성은 표본 2건으로 일반화 불가. → SGIS/용도지역 소스 확보 시 재개
  2. **투표자 거주 분포** — votes 스키마에 위치 정보 자체가 없음(계산할 데이터 부재). → 수집 설계 추가 전 보류
  3. **월 고정비 참고** — 공단 업종 평균 운영비의 실제 출처 확보 전 보류

## 창업 근거로서의 적합성과 보완 (v3 추가 → v4 감사 반영)

**역할 경계(확정 논지):** 이 리포트는 '후보 발굴 + 수요 검증' 도구다. 창업 최종 결정의 단독 근거가 아니며, 그렇게 포장하지 않는다. 수요 불확실성을 줄이는 것까지가 명당의 몫, 수익성·운영 판단은 창업자의 몫 — 이 분업을 리포트 안에서 명시하는 것이 신뢰의 원천.

**보완 4요소 — 근거 게이트 감사 반영:**
1. **수요 맥락화 ✅ 부분 유지** — "N명 = 개업 시 사용 확정 쿠폰 N×3,000원어치"(자체 데이터 산수)는 유지·프로토타입 반영됨. ~~투표자 반경 내 거주 분포~~ → **보류**(스키마에 위치 데이터 없음). 손익분기 비교는 매출 예측 영역이라 금지 유지.
2. ~~**월 고정비 참고**~~ → **보류** — 공단 통계 실출처 확보 후 재개.
3. ~~**지역 특성 노트**~~ → **보류** — 용도지역·인구 데이터 미확보, 인터뷰 표본 2건. 프로토타입에서 제거됨.
4. **"이 리포트가 답하지 않는 것" 체크리스트 ✅ 유지** — 데이터 주장이 없는 행동 안내라 근거 게이트 비대상. 프로토타입 반영됨.

**정직 문구 추가(확정 권고):** 선결제 투표 = 개업 시 1회 방문 의사의 증명이며 반복 구매의 보장이 아님을 대기 고객 카드 인근에 명시.

**실전 검증 계획:** 리포트를 장량동 사장 인터뷰이(솔밭샤브·카피엔드)에게 제시 → "창업 결정에 충분한가, 무엇이 더 필요한가" 확인. 결과를 리포트 개선과 발표(완성도)에 반영.

**예측 제공에 대한 입장 (3단계 로드맵 — 확정 논지):** 학습 데이터(개별 점포 실매출) 없는 매출 예측치는 근거 없는 숫자라 만들지 않는다. 카드 실거래 데이터를 가진 나이스비즈맵조차 개별 점포 예측이 아니라 관측 분포(평균/상위20%/하위20%)만 제시한다. 대신 정직한 경로 3단계:
1. **지금:** 관측 분포(업종·지역 평균, 출처) + 확정 수요(대기 N명 = 쿠폰 N×3,000원어치) 조합 — 예측이 아니라 관측+확정.
2. **지금 추가 가능 — 시나리오 계산기(팀 승인 대기):** 창업자가 객단가·재방문 주기를 입력하면 "대기 32명 × 월 2회 = 월 매출 기여 Z원"을 계산하는 위젯. 예측이 아닌 산수, 가정 주체 = 창업자. 프런트 구현 가능, 고유 데이터 기반 차별화.
3. **서비스 성숙 후:** 캠페인 누적으로 투표→실방문 전환율·쿠폰 회수율이 쌓이면 "투표 N명 → 첫 달 방문 M±k명" 예측이 근거를 갖고 가능. 우리 구조에서만 생기는 학습 데이터 = BM의 데이터 자산 논지와 연결. 발표에서 '향후 AI 확장'으로 제시.

심사 화법: "근거 없는 예측 대신 예측이 필요 없는 확정 수요를 제공하고, 데이터가 쌓이면 우리만의 전환율 예측 모델이 가능해진다."
