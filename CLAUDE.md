## 2. 담당 B 범위 · 원칙

데이터가 **나가는** 쪽: 3(적합도 산식)·4(겹침 해소 배치)·5(리포트)·6(캠페인·환불 로직).
- **엔진은 순수 함수.** DB 직접 접근 금지, 인자로 받은 것만 계산 → A와 독립, 더미로 테스트.

## 3. 확정된 것 (되돌리지 말 것)

### 3단계 — 수요 적합도 산식 ✅ 확정
```
score = demand × area_fit × competition_factor
  demand             = 직접 투표수 + w1 × 인근 공실(반경 500m)의 같은 업종 투표수
  area_fit           = 면적 범위 안 1.0, 벗어나면 비율 감점(하한 0.2)
  competition_factor = 동네 평균 대비 비율 기반:
                       ratio = 경쟁점수 / 그 업종 동네평균
                       ratio<=1 → 1.0(감점 없음) / ratio>1 → 1/(1+w2×(ratio-1))
```
- **성격 = 후보 필터.** 창업 결정 도구가 아니라 "이 자리를 후보로 볼지" 거르는 1차 신호. 진짜 창업 근거는 5단계 리포트 전체가 만든다. 산식이 매출·성공을 예측하는 척하면 과대약속(금지).
- **가중치 확정 + 근거(Q3-2 방어용):**
  - `w1 = 0.4` — 직접 표는 1차 신호, 인근 표는 절반 이하의 보조 신호
  - `w2 = 1.0` — 동네 평균 2배에서 수요 반토막(0.5). 정상 밀도는 존중, 명백한 포화만 감점
  - 인근 = 반경 500m
- 경쟁을 동네 평균 대비로 본 이유: "카페 3곳(흔함) ≠ 세탁소 3곳(드묾)". 절대 수로 감점하면 안 됨.
- 분해에 `competition_ratio`(동네 평균 대비 배수) 노출 — 5단계 포화 플래그가 이 값을 씀.

### 4단계 — 겹침 해소 배치 ✅ 확정
- 같은 업종을 두 공실에 배정 안 함 = 배정(assignment) 문제 → **헝가리안(scipy `linear_sum_assignment`) 채택.**
- 근거: 양덕동 공실이 수십 곳이어도 최적을 빠르게(전수탐색은 8곳에서 3.2초·폭발, 헝가리안은 300곳도 최적). "표준 최적화 알고리즘 사용" 방어.
- scipy 없으면 전수탐색(최적)+그리디 폴백으로 자동 동작 → 어느 환경에서도 실행.

### 리포트 = 배치 일치 ✅ 확정
- 리포트 1순위 추천 = `allocate` 배정 업종. 지도 핀과 리포트가 같은 업종을 말하게. (리포트가 공실별로 다시 점수 매기면 엔진이 없앤 겹침이 되살아나므로 금지.)

## 5. 다음 할 일 — 5단계 (이 순서로, 한 번에 하나씩)

리포트의 숙제 = "창업자에게 창업 근거를 정직하게 주느냐". 산식이 아니라 리포트 전체가 근거를 만든다. 무게중심은 우리만의 데이터 **"이미 지갑 연 대기 고객 N명"**.
1. **포화 플래그** — `competition_ratio`로 기회(≤1)/주의(1~2)/포화(≥2) 표시. 13번째 카페는 **veto가 아니라 "보여주되 경고"**(주민 유료 수요를 시스템이 무시하지 않는다는 우리 논지).
2. **경고를 위로** — 포화일 때 ① 한 줄 결론 옆에 같이 뜨게(스크롤 전에 보이게).
3. **산식 이원화 통일 결정** — 배치 랭킹(3요소 곱셈)과 리포트 표시 적합도%(4요소 가중합: 수요40/경쟁30/입지20/면적10)가 다름. 통일할지/분리 유지할지 사람에게 확인 후 진행.
4. **유동인구 구멍** — 창업자에게 크리티컬한데 소상공인 API에 없음. 정직하게 비울지/대체 지표로 갈지 사람에게 확인.

## 6. 미확정 · 열린 결정 (확정처럼 쓰지 말 것)

- 산식 이원화 통일 여부(위 5-3)
- 유동인구 데이터 소스(위 5-4) — SGIS/통신사 등 별도, MVP 필수 아님
- 캐시(모의) 모델: 교환비율(1캐시=1원?)·유효기간·재투표 시 현금 동일취급 여부
- `industries`에 **인허가(licenses) 컬럼 없음** — 리포트 ⑤에 필요. A 테이블에 추가 필요(현재 더미엔 임시로 넣어둠)
- 캠페인 기한·쿠폰 액면가(현재 30일·3,000원), 어뷰징 정책(MVP 밖)

## 7. 절대 규칙 (전 작업 공통)

- **결제는 모의**(플로우만, 실PG·사업자등록 없음). 환불 = **캐시 적립(모의)**.
- 용어: **투자·펀딩 금지 → 선결제·쿠폰 선구매.** 중개사는 경쟁자 아닌 **협력 파트너**.
- 시드 데이터는 전부 `is_seed=1`/`SEED` 표기(가상 데이터 지양 요건). 리포트 값엔 출처 태그(실측/API/예시/업종평균/파생).
- **미확정을 확정처럼 쓰지 않기. 출처 없는 수치 금지. 부재 증명은 "조사한 범위에서는"으로 한정.** 과장 금지.
- 지역 **하드코딩 금지, `region_code`로 파라미터화**(양덕동 첫 적용, 경북 확장).
- 통짜로 만들지 말 것. 단계별 만들고 검증 후 진행.
